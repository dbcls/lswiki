開催概要

  • 開催期間：2018年 1月 18（木）- 19日（金）
  • 開催場所：ライフサイエンス統合データベースセンター (DBCLS) @ 東京大学 柏の葉キャンパス駅前 サテライト 6階
  • アクセス：http://dbcls.rois.ac.jp/access
  • 開催連絡：http://groups.google.com/group/biohackathon-jp メーリングリストにて

プロジェクト

TPP グループ全体

H29〜

  • データサイエンスを加速させる微生物統合データベースの高度実用化開発 (黒川) - http://microbedb.jp/
    • ゲノム微生物学会MicrobeDB.jp講習会までのマイルストーン確認および更新（藤澤・岡別府）
  • 糖鎖科学ポータルの構築 (木下)
    • 糖タンパク質などでSPARQListを使う - https://sparqlist.glyconavi.org （山田）
      • ProtVistaを活用予定 - http://ebi-uniprot.github.io/ProtVista/developerGuide.html#adding-your-own-sources
    • せんでん：国際ワークショップ 3/5-9（ハッカソン:3/7-9） - http://glic.glycoinfo.org/meetings/LSworkshop2018/
  • 蛋白質構造データバンクのデータ検証高度化と統合化 (栗栖) - http://pdbj.org/
  • 個体ゲノム時代に向けた植物ゲノム情報解析基盤の構築 (田畑) - http://pgdbj.jp/
  • 疾患ヒトゲノム変異の生物学的機能注釈を目指した多階層オーミクスデータの統合 (菅野) - http://kero.hgc.jp/
  • エピゲノミクス統合データベースの開発と機能拡充 (沖) - http://chip-atlas.org/
  • ゲノム・疾患・医薬品のネットワークデータベース (金久) - http://www.kegg.jp/kegg/medicus/

継続中（〜H30）

  • プロテオーム統合データベース：jPOST (石濱) - http://jpostdb.org/
  • 生命動態システム科学のデータベースの統合化 (大浪)

〜H29

  • 生命と環境のフェノーム統合データベース (桝屋)
  • 生物種メタボロームモデル・データベースの構築 (有田)
  • 個別化医療に向けたヒトゲノムバリエーションデータベース (徳永)

SPARQLthon グループ

  • G2GML: BioHackathon 2017 で開始。LOD を Graph DB で利用するためのデータ変換。関連イベント 1/31 (千葉・山中)
  • ゲノムグラフ研究会 (大和田・原薗・横山)
    • 第1回研究会 2/27 第2回研究会 3/26
    • vcf2ggf
  • グラフ構築・解析エンジンの紹介（伊藤）
  • 文字修飾（上付き、下付き、イタリックなど）を表現・表示する際の方法の方針をきめたい。”科学では文字修飾が大事な意味を持っている（桝屋さん）”、化合物表記や糖鎖名称でも文字修飾があると助かります。（山田）
    • 参考： rdf:HTML Datatype - https://www.w3.org/TR/rdf11-concepts/#section-html
  • トリプルストア観測しました。(AllegroGraph 6.4.0、RDF4J 2.2.4、Stardog 5.2.0) AllegroGraph, RDF4JともにSERVICE句を利用したさいにリモートエンドポイントから得られるデータの型チェックを厳格にし始めている。このため、DBpediaの返すデータが弾かれるようになった。（山本）
  • うまかヤミー復活か。クローラー問題で長らく残念な状態でした。（佐藤、山口、山本）
  • LOD Surferメタデータクローラー更新されました。実行形式jarファイルもあるので、ダウンロードしてすぐに使えます。（小林さん、投稿山本）
  • SPARQL Builder API : 変更予定の GUI の仕様に合わせて一部仕様変更し API ver2としてデプロイ (山口)．
